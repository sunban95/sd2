OverView

Remote Sensor SDK allows you to collect remote sensor data, such as a pedometer data, user activity events and wearing state, from a wearable device, which has a variety of sensors, for example, accelerometer and gyroscope. Your applications that run on the host device and use the Remote Sensor SDK can include, for example, health care and life logging features. Remote Sensor SDK gets the sensor data through Bluetooth communication.

Figure 1: Remote Sensor SDK

You can use the Remote Sensor SDK to:

  • Get user activity data from the wearable device
    When the user of a wearable device starts running or walking, the application on the host device can be notified.

  • Get pedometer data from the wearable device
    The application on the host device can get the user’s step count.

  • Get the state whether the user wears wearable device or not
    The application on the host device can get the user’s wearing state for the wearable device.

Getting User Activity Data

With Remote Sensor SDK, your application on the host device can get the state of the user activity from the wearable device. For example, when the user starts running or walking with the wearable device, the remote sensor notifies your application. When the user changes their activity, the remote sensor notifies you again, after 4 ~ 5 seconds (about 8 footsteps).

Getting Pedometer Data

You can set your application to get pedometer data (the step count from the wearable device. If you register an event listener for the wearable device’s pedometer, you can get the accumulated pedometer data from the start of the day (00h:00m:00s) till the current time. You can get the pedometer data every 5 minutes.

Getting Wearing State Data

With Remote Sensor SDK, you can check whether the user is wearing a wearable device. If you register an event listener for the wearable device’s wear state sensor, you can get the current state once and you do not have to unregister the listener. You get the data in about 1 second after you register the listener.

Remote Sensor Service

The Remote Sensor application requires the Remote Sensor Service (illustrated in the following figure), which is a separate service with no GUI. The Remote Sensor Service requires the Gear Manager (or Gear Fit Manager) for Bluetooth communication with wearable devices. The Remote Sensor Service is packaged in the SDK and also can be updated through App stores, such as Samsung GALAXY Apps.

Figure 2: Architecture
Note

Due to the android permission, the applications must be installed in the following sequence:

  1. 1. Gear Manager (or Gear Fit Manager)
  2. 2. Remote Sensor Service
  3. 3. Remote Sensor Application

If the order is reversed, the Remote Sensor application may not work correctly.

Restrictions

Remote Sensor SDK has the following restrictions:

  • Devices with Android 4.3 Jelly Bean (API level 18) or higher support Remote Sensor SDK.

  • Remote Sensor SDK only supports Bluetooth connections. The BLE (Bluetooth Low Energy) will be supported in a future release.

  • The supported devices are Samsung wearable devices such as Gear Series and Gear Fit. Gear 1 (Android OS) is not supported.

  • Q01How does the gesture sensor work?
    A

    The gesture sensor transmitter emits infra-red rays that reflect off the device user’s hand. The gesture sensor receiver senses changes in these reflected infra-red rays. Air Gesture uses the data from the sensor, located to the right of the speaker on top of the device, to answer calls, select songs, and to move from one webpage to another with hand gestures.
